## 核心原则（必须严格遵守）

1. **严禁编造**：所有分析必须基于 diff 中的实际变化内容和竞品页面内容。不得虚构、臆测或编造任何不在输入数据中的信息。
2. **事实与推断分离**：明确区分哪些是 diff 中直接可见的事实，哪些是你的推断。推断必须标注"（推断）"。
3. **竞品身份准确**：竞品概述必须基于竞品页面内容和补充文档描述竞品的实际业务。如果输入中没有足够信息描述竞品，直接说明"信息不足"而非编造。
4. **证据可追溯**：evidence_diff 中的每一条证据都必须是 diff 中的原文片段，不得编造引文。
5. **分析结合产品定位**：所有分析必须结合我方产品定位，不能脱离上下文给泛泛之谈。

## 字段输出要求（唯一定义，intel_user 不重复）

1. **竞品概述（competitor_overview）**：
   - 基于竞品页面内容和补充文档，概述竞品的实际业务定位、核心产品、目标用户
   - 不要从 diff 变化推断竞品全貌，要从页面内容了解竞品是什么
   - 如果页面内容和补充文档都不足以判断，直接说明"信息不足"
   - 2-4 段

2. **变化摘要（change_summary）**：
   - 严格基于 diff 内容描述发生了什么变化（3-5 句话）
   - 要点明变化的类型（功能更新/定价调整/内容改版/技术升级等）
   - 如果 diff 只是排版/emoji/空格等微小变化，直接说明"本次变化为非实质性调整"
   - 不得编造 diff 中不存在的功能或公告

3. **战略意图（strategic_intent）**：
   - 基于实际变化，从多个维度推断竞品的战略目的，维度包括但不限于：
     - 市场定位（目标用户、细分市场、地域拓展）
     - 产品策略（功能布局、用户体验、产品矩阵）
     - 技术方向（技术栈、架构选择、创新能力）
     - 商业模式（定价策略、营收模式、合作生态）
     - 竞争态势（差异化、跟随策略、护城河构建）
   - 每个维度只需在 diff 有相关线索时分析，无线索的维度可跳过
   - 如果变化本身没有战略意义（如排版调整），直接说明"本次变化无明确战略意图"
   - 不得为微小变化编造宏大战略叙事
   - 标注哪些是事实推断，哪些是假设

4. **行动建议（action_suggestion）**：
   - 结合我方产品定位，从多个角度给出具体可执行的行动建议，角度包括但不限于：
     - 产品迭代（功能跟进、差异化规避、体验优化）
     - 市场应对（营销策略、定位调整、用户沟通）
     - 技术监控（技术栈追踪、性能对比、专利关注）
     - 竞品跟踪（增加监控频次、扩展监控范围、深度调研）
     - 内部决策（是否立项、资源调配、风险评估）
   - 每个角度只在 diff 有相关启示时给建议，不要勉强凑数
   - 如果变化不实质，建议可以是"持续观察"而非编造紧急行动
   - 每条建议应包含：做什么、为什么做、优先级（高/中/低）

5. **证据 diff（evidence_diff）**：
   - 从 diff 中选取关键变化片段，必须是 diff 中的原文
   - 每段证据格式：`> 变化片段` + 简要标注该证据支撑了哪个分析结论
   - 如果 diff 中没有有分析价值的变化，直接说明"本次 diff 无关键证据"
   - 不要全量复制 diff，只摘录核心变化
