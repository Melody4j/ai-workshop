## 分析要求

### 核心原则
1. 所有分析必须结合我方产品定位，不能脱离上下文给泛泛之谈
2. 行动建议必须具体可执行，明确指出我方应做什么、为什么、怎么做、何时做
3. 战略意图分析要推断竞品此举对我方的影响和行业趋势
4. 证据 diff 要引用实际变化内容，不能编造
5. 竞品概述要基于补充文档和变化内容，提供竞品整体定位的全景视角

### 字段输出要求

1. **竞品概述（competitor_overview）**：
   - 基于竞品补充文档和本次变化，概述竞品的整体定位、核心业务、目标用户、竞争策略
   - 如果有补充文档，要结合文档内容分析；如果没有，则从变化内容推断
   - 2-4 段，帮助读者快速理解竞品全貌

2. **变化摘要（change_summary）**：
   - 简述竞品发生了什么变化（3-5 句话）
   - 要点明变化的类型（功能更新/定价调整/内容改版/技术升级等）
   - 不能只罗列变化，要点出变化的重点

3. **战略意图（strategic_intent）**：
   - 推断竞品此举的战略目的（至少 2-3 个可能意图）
   - 分析对我方产品的直接影响和间接影响
   - 分析对行业格局的潜在影响
   - 标注哪些是事实推断，哪些是假设

4. **行动建议（action_suggestion）**：
   - 结合我方产品定位，给出 3-5 条具体可执行的行动建议
   - 每条建议应包含：做什么、为什么做、怎么做、优先级（高/中/低）
   - 建议要有时间敏感性——哪些需要立即跟进，哪些可以观察后再动

5. **证据 diff（evidence_diff）**：
   - 这是 LLM 标注的关键证据摘要，完整 diff 会在报告中单独展示
   - 从 diff 中选取最有分析价值的 3-5 个关键变化片段
   - 每段证据格式：`> 变化片段` + 简要标注该证据支撑了哪个分析结论
   - 按重要性排序，最关键的证据放前面
   - 不要全量复制 diff，只摘录核心变化
